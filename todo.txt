 Context                                                                                                           

 The app has hardcoded fixtures, canned chat responses, no real data tracking. We're building an LLM-powered
 proactive coach using LangGraph (agent framework), Mem0 (long-term memory via Supabase pgvector), and OpenRouter
 (LLM provider). The coach extracts exercise, nutrition, and lifestyle data from natural conversation, stores it,
 and feeds real data to Home + Trace screens.

         }
     }
 }

 mem0 = Memory.from_config(config)

 How Mem0 is used in the agent:
 1. Before each LLM call → mem0.search(user_message, user_id=user_id) retrieves relevant memories
 2. After each exchange → mem0.add([user_msg, assistant_msg], user_id=user_id) stores new facts
 3. Mem0 auto-extracts facts: "User did 22 pushups on April 21" → stored as a memory
 4. Mem0 auto-deduplicates and updates: "User's pushup PR is now 25" replaces old "PR is 22"

 backend/agent/nodes.py — Graph nodes

 async def load_context(state):
     """Pre-LLM node: fetch Mem0 memories + profile + recent summary"""
     memories = mem0.search(last_user_msg, user_id=state["user_id"])
     profile = await get_user_profile(state["user_id"])
     summary = await get_recent_summary(state["user_id"])
     return {"memories": format(memories), "user_profile": profile, "recent_summary": summary}

 async def call_model(state):
     """Main agent node: call OpenRouter LLM with tools"""
     system = build_system_prompt(state)
     messages = [SystemMessage(content=system)] + state["messages"]
     response = await llm_with_tools.ainvoke(messages)
     return {"messages": [response]}

 async def save_memory(state):
     """Post-LLM node: save exchange to Mem0"""
     last_user = get_last_user_msg(state)
     last_assistant = get_last_assistant_msg(state)
     mem0.add([
         {"role": "user", "content": last_user},
         {"role": "assistant", "content": last_assistant}
     ], user_id=state["user_id"])
     return state

 Step 4: API Routes

 backend/routes/chat.py

 POST /api/chat/stream — SSE streaming
 1. Auth: require Supabase JWT
 2. Save user message to chat_messages
 3. Run graph.astream() with stream_mode=["messages", "updates"]
 4. Stream tokens: data: {"type":"token","content":"word "}
 5. On tool execution: data: {"type":"extraction","logs":[...]}
 6. On done: save assistant message to chat_messages, emit data: {"type":"done"}

 GET /api/chat/history?before={cursor}&limit=20 — paginated history from chat_messages

 GET /api/chat/context — proactive opener
 - Check time of day, last activity, missing data
 - Short LLM call → return 1-line opener

 backend/routes/progress.py

 GET /api/progress/today → Home screen
 { "streak": 5, "today_plan": {...}, "insight": "...", "recent_logs": [...] }

 GET /api/progress/trace → Trace screen
 { "streak": 5, "best_streak": 8, "heatmap": [...], "prs": [...],
   "strength_trend": [...], "weekly_recap": "..." }

 GET /api/progress/chart?metric=pushups&period=30d → Line graphs
 { "points": [{"date":"2026-04-01","value":10}, ...], "unit": "reps", "label": "Pushups" }

 GET /api/progress/metrics → Available metrics for pills
 { "metrics": [{"key":"pushup","label":"Pushups","domain":"exercise"},
               {"key":"sleep","label":"Sleep","domain":"lifestyle"}, ...] }

 backend/routes/notifications.py

 POST /api/notifications/register — save OneSignal player_id to User
 Scheduled checks (cron or on-demand):
 - No exercise by noon → push "Morning session? Or saving it for tonight?"
 - No food by 2pm → push "What'd you have for lunch? I'm tracking."
 - No sleep logged by 9pm → push "How'd you sleep last night?"
 - Uses OneSignal REST API via httpx
